Studio Lab — Toolkit & Research

An agentic toolkit for teaching design judgment.

My students arrive able to make form but not yet to justify it. This is a growing kit of agentically-coded tools — built with Claude Code — that hand them the logics, language, and data to reason about a real site and the building they set on it: site forces, precedent, form, production, and the studio culture around the work. Used in a trust-but-verify stance — move fast, hunt the edge cases, stay the author.

Dev / demo only. The key is stored in this browser and sent straight from your browser to Anthropic (via the anthropic-dangerous-direct-browser-access header). Fine for trying the tools locally — but in a real deployment, route every model call through a small server so the key is never exposed to students.
How the integration works

Each AI tool calls one endpoint — POST https://api.anthropic.com/v1/messages — with a tool-specific system prompt. The browser hook:

fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: {
    "content-type": "application/json",
    "x-api-key": KEY,
    "anthropic-version": "2023-06-01",
    "anthropic-dangerous-direct-browser-access": "true"
  },
  body: JSON.stringify({
    model: "claude-opus-4-8",
    max_tokens: 1024,
    system: TOOL_SYSTEM_PROMPT,
    messages: [{ role: "user", content: studentInput }]
  })
})
// → data.content.find(b => b.type === "text").text

Notes on building the real site

This page is a single self-contained HTML file; the "database" is just a JavaScript array rendered into cards — easy to demo, easy to hand off. For the production version with a real DB, saved studio memory, peer-to-peer tracking, and LLM tools, the cleanest path is Next.js + Supabase on Vercel: content/tools live in a table (or typed config), and a serverless API route proxies Claude so the key stays server-side.

Lighter alternative: a static build (Astro / Eleventy) with content as Markdown/JSON, plus serverless functions for the model calls. Either way, the Tools / Research split here becomes two collections, and Work / Teaching drop in later as two more — one portfolio, four sections.
